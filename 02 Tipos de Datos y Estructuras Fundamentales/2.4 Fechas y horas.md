# Fechas y horas

MATLAB dispone de tres tipos para trabajar con tiempo: `datetime` (puntos en el tiempo), `duration` (duraciones exactas en tiempo) y `calendarDuration` (duraciones en unidades de calendario). Desde su introducción en R2014b, reemplazan las antiguas funciones de fecha como `datenum`.

## `datetime` — puntos en el tiempo

Representa momentos específicos en el tiempo, incluyendo zona horaria.

### Creación de datetimes

```matlab
% Fecha y hora actual
ahora = datetime('now')           % 2026-04-18 15:30:45
hoy = datetime('today')           % 2026-04-18

% Desde componentes
fecha1 = datetime(2026, 4, 18)                     % 2026-04-18
fecha2 = datetime(2026, 4, 18, 15, 30, 0)          % 2026-04-18 15:30:00

% Desde strings (automático)
fecha3 = datetime("2026-04-18")                    % 2026-04-18
fecha4 = datetime("18-Apr-2026 15:30:00")          % 2026-04-18 15:30:00

% Con formato específico
fecha5 = datetime("18/04/2026", 'InputFormat', 'dd/MM/yyyy')

% Arreglo de fechas
rango = datetime(2026, 4, 1, 1, 0, 0) + hours(0:3:24)  % cada 3 horas
```

### Formato de visualización

```matlab
t = datetime('now');

% Cambiar formato de visualización
t.Format = 'dd-MM-yyyy'           % "18-04-2026"
t.Format = 'MMMM d, yyyy'         % "April 18, 2026"
t.Format = 'HH:mm:ss'             % "15:30:45"

% Formatos comunes
default   = 'dd-MMM-yyyy HH:mm:ss'  % "18-Apr-2026 15:30:45"
iso_date  = 'yyyy-MM-dd'            % "2026-04-18"
us_date   = 'MM/dd/yyyy'            % "04/18/2026"
euro_date = 'dd/MM/yyyy'            % "18/04/2026"
```

### Extracción de componentes

```matlab
t = datetime(2026, 4, 18, 15, 30, 45);

year(t)    % 2026
month(t)   % 4
day(t)     % 18
hour(t)    % 15
minute(t)  % 30
second(t)  % 45

% Día de la semana
weekday(t)           % 7 (sábado, 1=domingo)
day(t, 'name')       % "Saturday"
day(t, 'shortname')  % "Sat"
```

### Zonas horarias

```matlab
% Con zona horaria
t_ny = datetime('now', 'TimeZone', 'America/New_York')
t_tokyo = datetime('now', 'TimeZone', 'Asia/Tokyo')

% Listar zonas disponibles
zones = timezones

% Cambiar zona
t_ny.TimeZone = 'Europe/London'
```

## `duration` — duraciones exactas

Representa diferencias de tiempo en unidades fijas (segundos, horas, etc.).

```matlab
% Creación
d1 = duration(1, 30, 0)           % 01:30:00 (1 hora, 30 minutos)
d2 = hours(3.5)                   % 03:30:00
d3 = minutes(90)                  % 01:30:00
d4 = seconds(3661)                % 01:01:01

% Formato
d1.Format = 'hh:mm:ss'            % "01:30:00"
d1.Format = 'hh:mm'               % "01:30"

% Operaciones
h = hours(2) + minutes(30)        % 02:30:00
dias = days(5)                    % 120:00:00
```

## `calendarDuration` — duraciones de calendario

Representa duraciones en unidades de calendario (años, meses, días) respetando irregularidades como meses de diferente duración.

```matlab
% Creación
cd1 = calyears(2) + calmonths(3) + caldays(5)   % 2y 3m 5d
cd2 = calmonths(15)               % 1y 3m

% Diferencia entre datetime
t1 = datetime(2026, 1, 31);
t2 = datetime(2026, 3, 1);
diff = between(t1, t2)            % 1m 1d (calendarDuration)

% Diferencia exacta
diff_exact = t2 - t1              % duration (en días exactos)
```

## Operaciones con fechas

### Suma y resta

```matlab
t = datetime(2026, 4, 18);

% Sumar duraciones
t + days(5)              % 2026-04-23
t + hours(48)            % 2026-04-20 00:00:00
t + calmonths(3)         % 2026-07-18 (respeta calendario)
t + calyears(1)          % 2027-04-18

% Restar fechas
t2 = datetime(2026, 5, 1);
diferencia = t2 - t      % duration (13 days)

% Diferencia en unidades específicas
days(t2 - t)             % 13
hours(t2 - t)            % 312
```

### Comparación

```matlab
t1 = datetime(2026, 4, 18);
t2 = datetime(2026, 4, 19);
t3 = datetime(2026, 4, 18);

t1 < t2      % true
t1 == t3     % true
t1 <= t2     % true
```

## Funciones útiles para arreglos de fechas

| Función | Descripción | Ejemplo |
|---------|-------------|---------|
| `dateshift` | Desplazar fechas a unidad específica | `dateshift(t, 'start', 'month')` → primer día del mes |
| `isbetween` | Verificar si fecha está en rango | `isbetween(t, t1, t2)` |
| `isweekend` | Verificar si es fin de semana | `isweekend(datetime('now'))` |
| `businessdate` | Días hábiles (Financial Toolbox) | `busdate(t, 'follow')` |

## Ejemplos prácticos

### Generar secuencias de fechas

```matlab
% Mensual
fechas = datetime(2026, 1, 1) : calmonths(1) : datetime(2026, 12, 1);

% Diario
dias = datetime(2026, 4, 1) : days(1) : datetime(2026, 4, 30);

% Cada 6 horas
horas = datetime(2026, 4, 18, 0, 0, 0) : hours(6) : datetime(2026, 4, 19, 18, 0, 0);
```

### Filtrar por fecha

```matlab
% Datos con fechas
fechas = datetime(2026, 4, 1) + days(0:30);
datos = rand(1, 31);

% Filtrar últimos 7 días
idx = fechas > datetime('today') - days(7);
datos_recientes = datos(idx);

% Filtrar por mes
idx_abril = month(fechas) == 4;
datos_abril = datos(idx_abril);
```

### Conversión entre tipos

```matlab
% datetime → string
s = string(datetime('now'))           % "2026-04-18 15:30:45"
s = datestr(datetime('now'))          % "18-Apr-2026 15:30:45"

% string → datetime
t = datetime("2026-04-18 15:30:45")

% numeric (días desde 0000-01-01) → datetime
t = datetime(738630, 'ConvertFrom', 'datenum')
```

## Consideraciones importantes

- `duration` vs `calendarDuration`: usar `duration` para tiempos exactos (horas, segundos), `calendarDuration` para operaciones que deben respetar calendario (sumar meses a una fecha)
- MATLAB usa el calendario gregoriano proléptico (se extiende hacia atrás)
- Las zonas horarias afectan cálculos con fechas históricas
- Para alta precisión, `datetime` tiene resolución de microsegundos
